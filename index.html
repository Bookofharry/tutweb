REST APIs Explained Simply:
The Middleman of the Web

🧵 You use REST APIs every single day. Most people don't even realize it. Let's change that today. 👇

Step 01 / Real-World Use

APIs Are Everywhere

REST APIs power almost the entire internet you use daily.

  • Checking the weather
  • Logging in with Google
  • Tracking Uber drivers
  • Loading Instagram feeds
Step 02 / The Analogy

The Restaurant Waiter

Let's use the simplest analogy possible: 🍽️ A Restaurant.

You're sitting at a table. The kitchen is where all the food is made. But you don't walk into the kitchen yourself. You tell the waiter what you want.

The waiter takes your request to the kitchen and brings back a response.

Step 03 / Web Translation

Client, Server, and API

In web development, the roles map exactly like this:

👤 You The Client

Sitting at the table, requesting resources.

🍳 Kitchen The Server

Where code executes and data is processed.

🍽️ Waiter The API

The middleman carrying requests and responses.

Step 04 / Core Definition

REST is a Set of Rules

REST is not a programming language. REST is not a framework.

REST is simply a standard set of rules and constraints for how clients and servers communicate with each other.

💡 It provides a standard way for applications to exchange data seamlessly.

Step 05 / HTTP Methods

HTTP Methods

REST APIs use HTTP Methods to execute actions. Think of these as instructions you're giving to the waiter:

Each method tells the server what action you want to perform. There are 4 you'll use constantly.

Step 06 / GET Method

📥 GET: Fetch Data

Used to request information without making modifications.

GET /api/v1/users GET /api/v1/weather/today

Analogy: You're asking for the menu. You want information — nothing is being changed.

Step 07 / POST Method

📤 POST: Create Data

Used to submit payloads and create new resources.

POST /api/v1/users (with body payload) POST /api/v1/orders

Analogy: You're placing your food order. Something new is being created.

Step 08 / Update Methods

✏️ PUT / PATCH: Update Data

Used to modify resources that already exist.

PATCH /api/v1/users/102 (update profile picture) PUT /api/v1/products/40 (replace details)

Analogy: Changing your order after you've placed it. Something exists, you're modifying it.

Step 09 / Delete Method

🗑️ DELETE: Remove Data

Used to destroy resources.

DELETE /api/v1/comments/98 DELETE /api/v1/users/102

Analogy: Canceling your order entirely. The resource gets removed.

Step 10 / The Format

JSON: The API Language

Servers respond with JavaScript Object Notation (JSON) payloads:

{ "name": "Harry", "role": "Software Engineer" }

JSON is a structured format for exchanging data. Think of it as the language most APIs speak.

Step 11 / Frontend vs Backend

Building Kitchen & Table

  • 🍳 Backend (Node, Express, Laravel)

    You're building the kitchen. You decide: what requests can come in? What responses go back?

  • 📱 Frontend (React, Next.js, Vanilla JS)

    You're sitting at the table. Making requests, waiting for responses, and rendering data.

Step 12 / The Complete Picture

The Complete Flow

1 Frontend

Sends HTTP request with params or bodies.

2 Waiter API

Receives and routes the request parameters.

3 Server Kitchen

Processes requests, interacts with databases.

4 JSON Response

Sends formatted data back to the interface.

SUMMARY

REST API Summary

  • REST API = The Waiter 🍽️
  • GET = Fetch Data 📥 • POST = Create Data 📤
  • PUT/PATCH = Update Data ✏️ • DELETE = Remove Data 🗑️
  • JSON = Data format spoken 📦
