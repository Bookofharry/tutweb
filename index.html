Node.js Modules:
The 3 Types You Need to Know

🧵 If Node.js modules confuse you, you're not alone. This concept took me longer than I'd like to admit to fully understand. Let's fix that today. 👇

Step 01 / The Imports

Where is it coming from?

When you start learning Node.js, one of the first confusing things you'll see is this:

const fs = require('fs') const express = require('express') const userController = require('./controllers/userController')

Everything is being imported. But where does it all exist? The answer is simple: there are only **3 types of modules in Node.js.**

Step 02 / Built-in

Built-in Modules

These come with Node.js itself.

No installation. No npm command. No setup. They are already available the moment Node is installed on your computer.

  • Examples of Built-in Tools

    📁 fs (File System) • 📂 path • 🌐 http • ⏰ timers

Step 03 / Built-in Example

Inside the Toolbox

const fs = require('fs')

You never ran npm install fs because Node already ships with it. Think of built-in modules as tools that come pre-packaged inside the Node.js toolbox.

Step 04 / Third-Party

Third-Party Modules

These are packages built by other developers in the community.

They are published to npm for anyone to use, and you must download and install them into your project yourself.

  • Popular Packages

    📦 Express • 📦 Mongoose • 📦 Dotenv • 📦 Axios • 📦 Nodemon

Step 05 / Installing packages

Installing from npm

# Install package npm install express # Import in code const express = require('express')

Someone else built it. You installed it. Now your project uses it.

Step 06 / node_modules

Don't Fear node_modules

Ever wondered what the massive node_modules folder actually is?

Many juniors are scared to touch it. Don't be. It's simply the storage home for your third-party packages. Everything installed through npm lives there. That's all.

Step 07 / Custom Modules

Custom Modules

These are the modules YOU create yourself.

As your project grows, keeping everything in one file becomes impossible. You split code into multiple files to stay organized. Those files become custom modules.

Step 08 / Exports & Imports

Custom Module Flow

// database.js function connectDB() { console.log('Database Connected') } module.exports = connectDB // main.js const connectDB = require('./database')

You built it, exported it, and imported it. That is the baseline custom module loop.

Step 09 / Code Architecture

Staying Organized

Instead of putting 5,000 lines in one single file, custom modules let you organize directories:

Split Clean Modules

routes • controllers • models • middleware • utilities

✅ Every file becomes its own modular component.

Step 10 / Mental Model

The Module Ecosystem

🏠 Built-in Node Created

fs, path, http, timers

📦 Third-Party Community Created

express, mongoose, dotenv, axios

🧠 Custom You Created

database.js, userController.js, auth.js

Step 11 / Benefit

Reading Code Faster

Once this mental model clicks, reading backend code becomes much easier. You'll instantly know:

  • "Did Node create this?"
  • "Did another developer create this?"
  • "Or did this project create it?"
SUMMARY

Node Modules Recap

There are only 3 types of Node.js modules:

  • Built-in Modules (Come with Node)
  • Third-Party Modules (Installed via npm)
  • Custom Modules (Created by you)
CONCLUSION

Master the Concepts

"Don't try to memorize commands. Master the concepts first."

Once you understand where modules come from, the structure of a Node.js application starts making perfect sense.
