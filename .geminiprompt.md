# Gemini Agent Prompt Guidelines

This file informs Gemini models on layout rules and code constraints for edits in this workspace.

## Objective
The codebase is a video-recording asset. The layout is structured as a vertical scroll card pipeline where each card functions as a "slide" in a social media video.

## Strict Guidelines for Gemini
- **Keep Elements Static**: Do not add Javascript click handlers to expand/collapse sections. The screen recorder reads text continuously while scrolling.
- **Large Fluid Typography**: Maintain maximum visibility with massive headers (`clamp(3rem, 7vw, 4.5rem)`) and readable body lines.
- **Vibrant Borders**: Every section card must be surrounded by a `4px` colored border representing its context (e.g. green for success/leverage, red for errors/warnings, purple/blue/black for steps).
- **Zero Script Bloat**: Keep `app.js` empty or as simple as possible to ensure fast local load times and no client console errors during recordings.
